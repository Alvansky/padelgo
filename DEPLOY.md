# Deployment Guide - Vercel + Supabase

## Prerequisites
- Vercel account
- Supabase account

## 1. Supabase Setup

### A. Create Project
1. Go to https://supabase.com
2. Create new project
3. Copy your Project URL and anon key from Settings > API

### B. Run SQL Schema
1. Open Supabase Dashboard > SQL Editor
2. Copy contents of `supabase/schema.sql`
3. Paste and run in SQL Editor
4. Verify tables: profiles, courts, bookings

### C. Enable Email Auth
1. Go to Authentication > Providers
2. Enable Email provider
3. Configure email templates (optional)

### D. Get First Admin User
After first registration via app:
```sql
UPDATE public.profiles SET role = 'admin' WHERE email = 'your-admin@email.com';
```

## 2. Vercel Setup (Hugo Frontend)

### A. Push to GitHub
```bash
git add .
git commit -m "Migrate to Supabase"
git push origin main
```

### B. Deploy on Vercel
1. Import repository in Vercel
2. Framework preset: Other (or Hugo)
3. Build command: `hugo --minify`
4. Output directory: `public`
5. Install Hugo extension or use Vercel's built-in Hugo support

### C. Set Environment Variables
In Vercel Dashboard > Settings > Environment Variables:
```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

## 3. Local Hugo Development

```bash
# Install Hugo
# Windows: choco install hugo-extended
# Mac: brew install hugo

# Run dev server
hugo server

# Build
hugo
```

For local testing with Supabase:
```bash
# Set env vars before building
set NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
set NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
hugo server
```

## 4. How It Works

### Frontend (Hugo on Vercel)
- Static HTML/CSS/JS generated by Hugo
- Supabase JS client loaded from CDN
- All auth and data operations via Supabase client
- RLS (Row Level Security) enforced by Supabase

### Backend (Supabase)
- **Auth**: Email/password authentication
- **Database**: PostgreSQL with RLS policies
- **Storage**: S3-compatible via Supabase Storage (if needed)

### Role Separation
- **User Dashboard**: Can only view own bookings
- **Admin Dashboard**: Can view all bookings, cancel/confirm, manage courts
- **Public Pages**: Courts listing, booking form (requires login to submit)

## 5. Key Routes

| Route | Access | Description |
|-------|--------|-------------|
| `/` | Public | Home page |
| `/order/` | Login required | Book a court |
| `/dashboard/` | User only | My bookings |
| `/admin/` | Admin only | Manage bookings/courts |
| `/login/` | Public | Login |
| `/register/` | Public | Register |
| `/settings/` | Login required | Account settings |

## 6. Troubleshooting

### "Supabase belum dikonfigurasi"
- Check Vercel env vars are set
- Redeploy after adding env vars

### Admin cannot access /admin/
```sql
-- Check role in Supabase SQL Editor
SELECT id, email, role FROM public.profiles;
-- Update if needed
UPDATE public.profiles SET role = 'admin' WHERE email = 'admin@example.com';
```

### Booking conflict not detected
- Check RLS policies are enabled
- Verify courts table has seeded data
- Check browser console for errors

### Dark mode not persisting
- localStorage is used, check browser settings
